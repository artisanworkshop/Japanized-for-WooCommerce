# Paidy 決済（後払い）

[← ドキュメントトップ](index.md)

---

## 概要

[Paidy](https://paidy.com/) は、クレジットカード不要で購入できる日本の後払い決済サービスです。顧客は購入時にメールアドレスと携帯電話番号だけで支払いができ、翌月まとめて支払う仕組みです。

> **対応通貨**: 日本円（JPY）のみ

---

## 有効化・初期設定（オンボーディングウィザード）

### 1. Paidy のアカウント取得

[Paidy 加盟店申込ページ](https://paidy.com/merchant/) からアカウントを申請し、API キーを取得します。

### 2. テスト環境での設定

1. **WooCommerce → 設定 → 決済** から「Paidy」を選択します。
2. 環境を「テスト」に設定します。
3. テスト用の公開鍵・秘密鍵を入力します。
4. 設定ウィザードに従って動作確認を行います。

### 3. 本番環境への切り替え

テストが完了したら、環境を「本番」に切り替え、本番用の API キーを入力します。

---

## 設定項目

**WooCommerce → 設定 → 決済 → Paidy** で以下を設定します。

| 設定項目 | 説明 |
|----------|------|
| 有効化 | 決済方法を有効/無効にします |
| 環境 | テスト / 本番環境の選択 |
| 公開鍵（本番） | Paidy から発行された本番用公開 API キー |
| 秘密鍵（本番） | Paidy から発行された本番用秘密 API キー |
| 公開鍵（テスト） | テスト用公開 API キー |
| 秘密鍵（テスト） | テスト用秘密 API キー |
| ストア名 | Paidy 決済画面に表示されるショップ名 |
| ロゴ画像 URL | Paidy 決済画面に表示されるロゴ画像の URL |
| 説明文 | チェックアウト画面での Paidy の説明文 |
| 注文ボタンテキスト | 注文確定ボタンのテキスト |
| デバッグモード | デバッグログを記録する（開発時のみ使用） |

---

## 動作フロー

1. 顧客がチェックアウトで「Paidy」を選択して注文
2. Paidy の認証画面が表示される（メールアドレス・携帯番号入力・SMS 認証）
3. 認証完了後、注文が確定する
4. 顧客は翌月にコンビニ・銀行振込・口座振替で支払い

---

## ブロックチェックアウト対応

WooCommerce ブロックチェックアウトにも対応しており、追加設定なしで利用できます。

---

## API キーが未設定の場合

公開鍵・秘密鍵が設定されていない場合、チェックアウト画面に Paidy は表示されません。必ず両方の API キーを設定してから有効化してください。

---

## テスト方法

Paidy はテスト環境用の API キーとテスト用の電話番号・認証コードを提供しています。テスト方法の詳細は [Paidy 開発者ドキュメント](https://paidy.com/docs/) をご参照ください。
